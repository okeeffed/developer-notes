## Queries with Joins and Aggregations

Joins:

- Produces values by merging together rows from different related tables.
- Use a join _most times_ that you're asked to find data that involves multiple resources.

Aggregation:

- Looks at many rows and calculates a single value.
- Words like 'most', 'average', 'least' are a sign that you need to use an aggregation.

## Joining Data From Different Tables

"For each comment, show the contents of the comment and the username of the user who wrote the comment."

```sql
SELECT contents, username FROM comments JOIN users ON comments.user_id = users.id;
```

## Alternative forms of syntax

- The order of the `FROM` and `JOIN` clauses is important.
- We must give context if column names collide.
- Tables can be renamed using the `AS` keyword.
- There are a few kinds of joins!

## Missing Data in Joins

This is an example of the order for `FROM` and `JOIN` being important.

If we have a photo with no user, we can't show the username. We will have missing data for that entire row. That means we are now not listing all of our photos.

It is not joining because our `ON` clause is not satisfied.

## Four Kinds of Joins

| Join Type            | Keyword    | Description                                          |
| -------------------- | ---------- | ---------------------------------------------------- |
| Inner join (default) | INNER JOIN | Joins rows that have matching values in both tables. |
| Left outer join      | LEFT JOIN  | We do not throw away rows from source table.         |
| Right outer join     | RIGHT JOIN | We do not throw away rows from joined table.         |
| Full outer join      | FULL JOIN  | We do not throw away rows from either table.         |

## Does Order Matter?

Yes it does.

The example was given of changing the order of the tables and in one case we got a photo without a user and in the the other we got the user without a photo but both were DIFFERENT records.

## Where with Join

"Users can comment on photos that they posted. List the url and contents for every photo/comment where this occured."

```sql
SELECT contents, url
FROM comments
JOIN photos ON comments.photo_id = photos.id
WHERE comments.user_id = photos.user_id;
```

## Three Way Joins

We can merge three different tables. Conceptually this is difficult, but the SQL is straight forward enough.

"List the url, contents and username for every photo/comment where this occured."

```sql
SELECT contents, url, username
FROM comments
JOIN photos
ON comments.photo_id = photos.id
JOIN users
ON photos.user_id = users.id AND comments.user_id = users.id
WHERE comments.user_id = photos.user_id;
```

Note: when joining the third table, it is worth noting that the third join will be more complicated with the `ON` clause.
